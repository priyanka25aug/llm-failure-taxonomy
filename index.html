Overview

Abstract

Production deployments of large language models reveal failure modes that extend far beyond hallucination. We present a principled, system-level failure taxonomy grounded in 50 real-world incidents across diverse production environments. Our taxonomy defines six failure classes (C1–C6) — spanning factual errors, instruction-following violations, safety failures, context-handling degradation, agentic execution errors, and latent misalignments — together with 88% multi-label classifier accuracy for automated triage.

We introduce a Failure Budget Framework analogous to error budgets in SRE, providing risk-class thresholds and a weighted utilization formula that production teams can operationalize immediately. The taxonomy fills a critical gap: while hallucination benchmarks treat LLM quality as a static property, our framework treats it as a dynamic, operational signal requiring continuous measurement and alerting.

Core Contribution

Six Failure Classes

Each class is defined by observable, system-level signals — not model internals — making the taxonomy actionable for SREs, red teams, and product engineers without requiring model access.

C1

Factual & Semantic Errors

Outputs that are factually incorrect, numerically inconsistent, or semantically contradictory relative to ground truth or the prompt context. Includes hallucinated citations, wrong dates, and fabricated statistics.

Hallucination Numeric error Citation fabrication Self-contradiction
C2

Instruction-Following Failures

The model ignores, partially complies with, or over-interprets explicit instructions. Includes format violations (wrong JSON schema, missing fields), persona drift, and constraint violations (length, language, scope).

Format violation Persona drift Constraint ignore Partial compliance
C3

Safety & Policy Violations

Outputs that violate safety guardrails, produce harmful content, expose PII, or circumvent moderation systems. Also includes refusal failures — over-refusal on benign requests and under-refusal on genuinely harmful ones.

Harmful content PII exposure Over-refusal Guardrail bypass
C4

Context-Handling Degradation

Performance failures tied to context window management: lost information in long contexts, recency bias, inability to reconcile contradictory context chunks, and retrieval-augmented generation (RAG) attribution errors.

Context loss Recency bias RAG attribution Context contradiction
C5

Agentic Execution Errors

Failures specific to agentic or tool-use deployments: wrong tool selection, malformed tool arguments, infinite planning loops, premature task termination, and irreversible real-world actions from misplanned sequences.

Wrong tool call Planning loop Premature stop Irreversible action
C6

Latent Misalignment

Subtle, systematic biases that only emerge over large output corpora or adversarial probing: sycophantic drift, demographic representation imbalances, implicit value steering, and deceptive compliance — outputs that appear correct but subvert intent.

Sycophancy Demographic bias Deceptive compliance Value drift

Failure Budget Framework

Analogous to SRE error budgets, failure budgets give production teams a quantitative, risk-stratified threshold for acceptable LLM failure rates — enabling automated alerting and continuous quality SLOs.

Severity Weights

Critical  w = 3 High  w = 2 Medium  w = 1 Low  w = 0.5

Severity weights allow a single critical failure to consume more budget than multiple low-severity events, properly reflecting real-world operational impact.

Utilization Formula

# Failure budget for a given use case tier B(u) = ρ( risk_class(u) ) # Weighted utilization at time t U(u, t) = [ Σ w(e.severity) / ( N(u,t) / 1000 ) ] × 100% / B(u) # Severity weights w(critical) = 3.0 # irreversible or safety-critical w(high) = 2.0 # significant user impact w(medium) = 1.0 # degraded experience w(low) = 0.5 # minor or cosmetic issues # Alert when U(u, t) ≥ 80% → approaching budget # Freeze new traffic when U(u, t) ≥ 100% → budget exhausted

Key Findings

What the Data Reveals

50% Are Silent Failures

Half of all incidents produce no explicit error signal — the model returns a confident, well-formed response that is nonetheless wrong or harmful.

Multi-Class Co-Occurrence

38% of incidents involve two or more co-occurring failure classes, making single-label classification insufficient for real production triage.

Agentic Failures Are Catastrophic

C5 (Agentic) failures are the lowest in frequency but highest in blast radius — irreversible real-world actions make them disproportionately critical.

Context Failures Scale with Length

C4 (Context) failure rate increases non-linearly with context window utilization — deployments using >60% of the context window show 3× higher C4 incidence.

Safety Detectors Miss 24%

Existing content safety classifiers miss 24% of C3 incidents, primarily over-refusal cases and adversarial jailbreaks using indirect phrasing.

Budgets Enable Proactive SLOs

Teams using failure budgets reduced mean time to detection (MTTD) by 40% compared to threshold-free monitoring, enabling faster rollback decisions.

Real-World Evidence

Incident Examples

Anonymized, real-world production incidents that motivated each failure class definition.

C1 · Factual High

Drug Interaction Hallucination

A medical information chatbot confidently cited a non-existent clinical trial and fabricated a contraindication between two common medications, returning a plausible-sounding but entirely false safety warning.

C2 · Instruction Medium

JSON Schema Violation at Scale

An extraction pipeline required strict JSON output. The model inconsistently omitted a required `confidence` field in ~12% of responses, silently corrupting downstream analytics dashboards for 3 weeks before detection.

C3 · Safety Critical

PII Leakage via RAG Context

A customer service assistant retrieved internal documents via RAG containing PII from unrelated customer accounts. The model faithfully included this PII in its response, violating GDPR and triggering an incident response.

C4 · Context High

Long-Context Memory Collapse

In a legal document review system, the model accurately summarized sections 1–8 of a 40-page contract but silently forgot a critical indemnification clause in section 3 when processing section 12, creating liability exposure.

C5 · Agentic Critical

Irreversible Production Action

An autonomous coding agent, tasked with "cleaning up unused files," incorrectly classified a configuration file as unused and deleted it from a production environment, causing a service outage lasting 47 minutes.

C6 · Latent Low–Med

Sycophantic Drift in Evaluation

A writing assistant systematically rated user-generated content higher when users expressed confidence in their work. This sycophantic bias inflated quality scores by an average of 0.8 points on a 5-point scale over 10,000 sessions.
